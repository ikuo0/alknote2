
- 基盤ルール
  - アカウントID
    - 書式 "aid.{UNIXTIME(ms)}.{UUID4}.{UUID4}"
    - 保存先 DynamoDB
    - アカウントID文字列のみで認証完了できる
    - １週間に１回マジックリンク検証を要求し本人検証を行う
  - 検証トークン
    - 書式 "vtoken.{UNIXTIME(ms)}.{UUID4}"
    - マジックリンクに付与してメール送信するトークン
    - 有効期限 １時間
    - 3回間違えたら検証トークンは使用不能となる
    - レコード削除はDynamoDBのTTL機能で自動削除する
  - インスタンス
    - インスタンスID書式 "iid.{UNIXTIME(ms)}.{UUID4}"
    - 保存先 DynamoDB
    - 投稿やノートなどのリソースを保持する単位
    - インスタンスIDはアカウントIDに紐づく
  - アクセストークン
    - 書式 "atoken.{UNIXTIME(ms)}.{UUID4}"
    - 保存先 DynamoDB
    - アクセストークンは インスタンスID と 権限セット を保持する
    - 権限種類
      - 投稿
      - 編集（削除含む）
      - 閲覧
      - エクスポート
      - 共有（トークン発行）
      - 管理（全削除、無効化、課金管理）
    - 権限セット
      - オーナー: 全権限
      - 共有投稿者: 投稿、閲覧、エクスポート、共有 の権限を持つ
      - 投稿者: 投稿、閲覧、エクスポート の権限を持つ
    - 課金猶予期間について
      - 課金有効期限切れ後に１週間の猶予期間を設ける
      - 課金猶予期間中は「投稿」権限を無効とする、その他の権限は維持する
      - 課金猶予期間中もオーナーの週一の本人検証は要求する
    - 発行種類
      - オーナーアクセストークン
        - アカウントID認証により発行されるアクセストークン
        - オーナー 権限セットを持つ
      - ゲストアクセストークン
        - オーナー、又は共有投稿者のアクセストークン認証により発行可能なアクセストークン
        - 共有投稿者 又は 投稿者 権限セットを持つ、発行時に選択する
    - 期限、再認証の要求
      - オーナーアクセストークンは１週間に一度マジックリンクによる本人検証を要する
      - オーナー以外のアクセストークンは本人検証を要求しない、無期限有効
      - オーナー以外のアクセストークンはオーナー権限により無効化、削除が可能
  - 投稿データ
    - テキスト
      - DynamoDBに保存
      - アクセストークン認証により投稿可能
      - アクセストークン、時刻、GPS情報、IPアドレス住所情報 を保存、取得できない情報はデータが無い旨をフラグセットし初期値で埋めた状態で保存
      - テキスト, 最大10,000文字
    - 画像
      - S3 に保存
      - DynamoDB にメタデータ保存
      - アクセストークン認証により投稿可能
      - アクセストークン、時刻、GPS情報、IPアドレス住所情報 を保存、取得できない情報はデータが無い旨をフラグセットし初期値で埋めた状態で保存
      - サイズはフルハイ（1920x1080）に収まるように長い辺を基準にリサイズ
      - 元画像がなにであっても quality 75 でJPEG画像へと変換
  - 閲覧
    - アクセストークン認証により閲覧可能
    - アクセストークンに紐づいた instance_id の投稿のみ閲覧可能
    - 時系列で最新100件を取得可能
  - 課金
    - Stripe を利用して課金管理を行う
    - ３５０円で１か月利用期間が延長される
    - 課金開始日時(billing_utms)に加算する、課金日時に加算ではない点に注意
    - 課金有効期限切れ後に１週間の猶予期間を設ける
    - 課金猶予期間中は「投稿」権限を無効とする、その他の権限は維持する
    - 課金猶予期間中もオーナーの週一の本人検証は要求する

- 以下に要件定義に必要なバックエンド機能を列挙する
- ユースケース（バックエンド機能）
  - サインアップ
    - メールアドレス検証
      - メールアドレス、パスワード（数字４桁）を受け取りマジックリンクを送信
    - アカウント発行
      - マジックリンククリックでアカウント発行ページへ遷移
      - パスワードを入力してアカウント発行
  - アクセストークンの発行
    - オーナーアクセストークン発行
      - アカウントID を受け取りデータが存在すれば認証成功
      - 課金期限切れの場合は投稿権限を無効化するが、その他の権限は維持する
      - 最後の本人検証から１週間以上経過している場合は認証失敗とし、「再認証要求エラー」を返す
    - ゲストアクセストークン発行（共有投稿者、投稿者）
      - 共有権限を持つアクセストークン認証により発行可能
      - 共有投稿者 又は 投稿者 権限セットを選択できる
  - サインイン
    - アクセストークン認証
      - アクセストークンがレコードに存在し、かつ有効であれば認証成功、オーナーは本人検証の期限も確認する
  - 投稿
    - テキスト投稿
      - アクセストークン認証により投稿可能
    - 画像投稿
      - アクセストークン認証により投稿可能
  - 閲覧
    - アクセストークン認証により閲覧可能
  - 課金
    - 本人検証
      - オーナーアクセストークン認証により本人検証を行う
    - 課金処理
      - 検証トークンによる認証を行い、Stripe API を利用して課金処理を行う
